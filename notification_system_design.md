# Notification System Design

## Stage 1
Created APIs to send and receive notifications.

## Stage 2
Used Users and Notifications tables to store data.

## Stage 3
Used indexing and optimized queries for faster retrieval.

## Stage 4
Used Redis caching to reduce database load.

## Stage 5
Used a message queue for reliable notification delivery.

## Stage 6
Used a Priority Queue to keep the top 10 important notifications.
Priority:
- Placement = High
- Result = Medium
- Event = Low
